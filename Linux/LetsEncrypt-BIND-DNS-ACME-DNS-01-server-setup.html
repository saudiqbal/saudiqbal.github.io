LetsEncrypt BIND DNS and ACME DNS-01 server setup guide

Are you looking to setup your own DNS server for LetsEncrypt's ACME DNS-01 verification challenges then this guide is for you. I use Debian Linux so this guide is based on Debian 12 at the time of this writing.

Domain registrar DNS records setup

First add a new DNS record for your dns server, for example dns.example.com AAAA 2002:5d63:a55b:42df:5d01:2359:a67e:737d A/AAAA record with your server IP where you will serve your BIND9 DNS server.

Server Setup

Install BIND9 DNS server apt install bind9

Now generate a key to update your DNS server tsig-keygen -a hmac-sha256 example, I chose example for my primary domain example.com and www.example.com. You can generate as many keys as you want for each of your servers, this is good because just in case one of your servers gets compromized then all you have to do is replace the key in your DNS server.

Now we start configuring BIND9 server. This example includes for different servers using the DNS server for generating LetsEncrypt certificates

Now configure named.conf.options file, replace the example IP with your own, for IPv4 use listen-on { 125.125.125.125; };, I only use IPv6 so mine is set to listen-on-v6 { 2002:5d63:a55b:42df:5d01:2359:a67e:737d; };

Now create a new zone file listed above (/var/lib/bind/example.com.zone), replace the values from your domain registrar.

Now restart BIND server with your new settings systemctl restart bind9

Client server setup

Install apt install dnsutils

Now create a new file for each server you created above and paste the exact contant in the file, for example your primary client, create a new file in /root/acme.example.key

For each new server paste the content of key file you created above, server1, server2, server3 and so on.

Now open the terminal in your client server and run (export) these commands.

Now finally request the certificate using acme.sh
